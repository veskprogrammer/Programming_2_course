## 1. Постановка задачи

Разработать асинхронное веб-приложение на Python для отслеживания актуальных курсов валют.

В лабораторной работе необходимо реализовать REST API с использованием:

1. `FastAPI` — для создания веб-приложения и эндпоинтов.
2. `SQLAlchemy` — для работы с базой данных через ORM.
3. `SQLite` — в качестве локальной базы данных.
4. `httpx` — для асинхронного запроса к сайту Центрального банка РФ.
5. `Pydantic` — для валидации входных и выходных данных.

**Требования:**

* Использование аннотации типов.
* Наличие документации в коде — `Docstrings` в стиле PEP-257.
* Асинхронные эндпоинты FastAPI.
* Асинхронная работа с базой данных через `sqlalchemy.ext.asyncio`.
* Реализация CRUD-операций для пользователей.
* Реализация подписок пользователей на валюты.
* Получение списка валют и курсов с сайта ЦБ РФ.
* Корректная обработка ошибок с HTTP-статусами `404` и `409`.

### Модуль `database.py`

В данном модуле выполняется настройка подключения к базе данных SQLite.

Используется асинхронный драйвер `aiosqlite` и асинхронная сессия SQLAlchemy.

```python
DATABASE_URL: str = "sqlite+aiosqlite:///./currency_api.db"
```

Функция `get_db()` используется как dependency в FastAPI и передает сессию базы данных в эндпоинты.

---

### ORM-модели

В проекте реализованы три основные модели.

#### Модель `User`

Хранит информацию о пользователе:

* `id` — первичный ключ;
* `username` — уникальное имя пользователя;
* `email` — уникальная электронная почта;
* `created_at` — дата создания пользователя.

#### Модель `Currency`

Хранит информацию о валюте:

* `id` — первичный ключ;
* `code` — буквенный код валюты, например `USD`;
* `name` — название валюты;
* `rate` — последний сохранённый курс валюты.

#### Модель `Subscription`

Связующая таблица между пользователями и валютами.

Она реализует отношение many-to-many: один пользователь может подписаться на несколько валют, и на одну валюту может быть подписано несколько пользователей.

Для защиты от повторной подписки используется уникальная пара:

```python
UniqueConstraint("user_id", "currency_id", name="unique_user_currency")
```

---

### Pydantic-схемы

Для проверки и описания данных используются Pydantic-схемы.

В проекте есть схемы:

* `UserCreate` — создание пользователя;
* `UserUpdate` — обновление пользователя;
* `UserRead` — вывод пользователя;
* `UserWithSubscriptions` — вывод пользователя со списком подписок;
* `CurrencyRead` — вывод валюты;
* `CurrencyRate` — вывод курса валюты;
* `SubscriptionCreate` — создание или удаление подписки;
* `SubscriptionRead` — результат операции с подпиской.

---

### Получение курсов валют с сайта ЦБ РФ

Для получения актуальных курсов используется сервис `cbr.py`.

Запрос выполняется по адресу:

```text
https://www.cbr.ru/scripts/XML_daily.asp
```

Для запроса используется библиотека `httpx`, а для разбора XML — стандартная библиотека `xml.etree.ElementTree`.

Курс приводится к значению за одну единицу валюты. Например, если курс указан за 10 или 100 единиц, значение делится на `Nominal`.

## 3. Основные эндпоинты

### Пользователи

#### `POST /users/`

Создание нового пользователя.

Пример тела запроса:

```json
{
  "username": "georgiy",
  "email": "georgiy@example.com"
}
```

#### `GET /users/`

Получение списка всех пользователей.

#### `GET /users/{user_id}`

Получение информации о конкретном пользователе, включая список валют, на которые он подписан.

#### `PUT /users/{user_id}`

Обновление информации о пользователе.

Пример тела запроса:

```json
{
  "email": "new_email@example.com"
}
```

#### `DELETE /users/{user_id}`

Удаление пользователя.

---

### Валюты

#### `GET /currencies/`

Получение списка валют из локальной базы данных.

#### `POST /currencies/update`

Обновление списка валют и курсов с сайта Центрального банка РФ.

Этот эндпоинт нужно вызвать перед созданием подписок, чтобы в базе появились валюты.

#### `GET /currencies/{currency_code}/rate`

Получение текущего сохранённого курса валюты.

Пример:

```text
GET /currencies/USD/rate
```

---

### Подписки

#### `POST /subscriptions/`

Подписка пользователя на валюту.

Пример тела запроса:

```json
{
  "user_id": 1,
  "currency_code": "USD"
}
```

#### `DELETE /subscriptions/`

Удаление подписки пользователя на валюту.

Пример тела запроса:

```json
{
  "user_id": 1,
  "currency_code": "USD"
}
```

### Получение валют с ЦБ РФ

```python
async def fetch_currencies_from_cbr() -> list[dict[str, str | float]]:
    """Получить список валют и курсов с сайта ЦБ РФ."""
    async with httpx.AsyncClient(timeout=15) as client:
        response = await client.get(CBR_URL)
        response.raise_for_status()
```

### Создание пользователя

```python
@router.post("/", response_model=UserRead, status_code=status.HTTP_201_CREATED)
async def create_user(user_data: UserCreate, db: AsyncSession = Depends(get_db)) -> User:
    """Создать нового пользователя."""
    user = User(username=user_data.username, email=user_data.email)
    db.add(user)
    await db.commit()
    await db.refresh(user)
    return user
```

### Создание подписки

```python
@router.post("/", response_model=SubscriptionRead, status_code=status.HTTP_201_CREATED)
async def create_subscription(
    data: SubscriptionCreate,
    db: AsyncSession = Depends(get_db),
) -> SubscriptionRead:
    """Создать подписку пользователя на валюту."""
```

## 7. Обработка ошибок

В приложении реализована базовая обработка ошибок.

Примеры:

* `404 Not Found` — пользователь, валюта или подписка не найдены.
* `409 Conflict` — попытка создать пользователя с уже существующим email или username.
* `409 Conflict` — попытка повторно подписать пользователя на одну и ту же валюту.
